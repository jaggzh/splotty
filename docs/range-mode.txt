## Background:
Text-based serial data plotter for effectively unlimited number-of-fields serial/data plots, allowing tabular data as plain numerical, or labeled like arduino:#.##  lbl2:#  etc.
The plotter also supports field groups (named-collections defined in a yaml), presently used to toggle visibility.

Being text-based, I only implemented vertical scrolling.
At present, the entire plot just auto-ranges to the min/max plots. (tbh I didn't even look at how it was implemented; the llm might have followed some idea I had on a very slow moving assymetric min/max (fast growth, slow shrink).


## New bounds feature(s) - General idea:
A. Need a toggle to normalize all plots.
B. I need to know some methods used for setting ranges for the scaling/bounds of the plot. These will be toggled with a prefix hotkey (r)anges, perhaps like r-0 for all; we need to keep in mind that whatever fixed letters we use, we're consuming one that might be used for existing plots or field-groups.

I need to know, in the industry, what methods are used for actually handling the range calculations, adjustments, and groupings, to know "what others have done" (and then do it right). (As an example, Arduino's just normalizes instantly to the min/max of whatever data is in the view (and it's awfully limited as a result, and awkwardly flickers the scale); yet, slow-shrinking ranges are confusing as the plot scale "glides" and you visually see the plot changing, but it's hard to differentiate if it's the plot or the ranges doing it.
